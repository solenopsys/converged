
# Процессор узлов 

Это утилитарный тупейший компонент как молоток его задача забивать гвозди, он вообще
не знает ни очем во круг его задача просто в ответ на запрос исполнять код ноды и оповещать о результатах и держать await до таймаута все. Он не знает даже ничего про контексты и не знает в рамках какого контекста исполняется. 

При том все его узлы уже инициализированы провайдерами ему об этом заботится не нужно. 

- При старет все коды всех воркфлоу заливаются в мап с ключами по именам workflowName:nodeCodeName (просто через параметры)
- Процессор при старте сбрасывает все зависжише состояния в error - так как он единсвенный исполнитель и он только что запустился. 
- Процессор просто создает объект состояния в базе и возвращает id 
- Процессор вовращает события ошибки и завершения

 ## Состояние узла (данные)
   id - uuid7
   code - workflowName:nodeCodeName
   state - new, start, end, err
   data - inputData
   result - outputData
