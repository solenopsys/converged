# Files Engine

Назначение
- простое key-value хранилище на файловой системе.
- ключ — относительный путь файла внутри `dataLocation`.

Backend
- файловая система.

Ключи и данные
- ключ может содержать поддиректории; директории создаются автоматически при записи.
- значение — `Uint8Array` (сырые байты).
- хеширование и content-addressed логика находятся на уровне сервиса, не в engine.

API (основное)
- `open`, `close`, `migrate`
- `put`, `get`, `exists`, `delete`, `listKeys`, `getStats`

Миграции
- через общий механизм `Migration`/`Migrator`.

Файлы
- `dataLocation` — корневая папка хранения.
